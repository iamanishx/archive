Attention Is All You Need — Interactive Paper

Vaswani et al., 2017 · arXiv:1706.03762

Abstract

Before 2017, the best sequence models — for tasks like machine translation — were built on recurrent or convolutional neural networks. These worked, but had a fundamental bottleneck: they processed tokens sequentially, one position at a time. The Transformer throws that out entirely.

The core claim

A model built solely on attention mechanisms — no recurrence, no convolution — is not only possible, but superior in quality, faster to train, and more parallelisable.

1 Introduction

RNNs and LSTMs were the dominant paradigm. They generate a hidden state hₜ from the previous state hₜ₋₁ and the current input. This sequential dependency is the problem.

Problems with RNNs

— Cannot parallelise within a training example

— Memory limits batching at long sequences

— Path length for long-range deps = O(n)

Transformer's fix

— All positions processed in parallel

— Any two tokens are one hop apart

— Trained in 12h on 8 P100s (base model)

Attention mechanisms already existed — but were always used alongside a recurrent network. This paper is the first to rely on attention entirely, replacing the recurrent layers.

3 Model architecture

The Transformer is an encoder-decoder model. Click through each layer to understand what it does.

Both encoder and decoder use residual connections: output = LayerNorm(x + Sublayer(x)). This helps gradients flow during training.

3.2 Scaled dot-product attention

Attention maps a query against a set of key-value pairs. The output is a weighted sum of the values, where weights come from how well the query matches each key.

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V

Why the √d_k scaling?

For large d_k, dot products grow large in magnitude, pushing softmax into near-zero-gradient regions. Dividing by √d_k keeps gradients healthy. Without it, additive attention outperforms dot-product at large d_k.

Interactive attention demo

Click a token to see how much it attends to every other token.

Masking in the decoder

Positions can't attend to future positions. Those entries are set to −∞ before softmax → weight becomes 0. This preserves the auto-regressive property during generation.

3.2.2 Multi-head attention

Instead of one big attention function, run h = 8 parallel heads, each with its own learned projections down to d_k = 64. Concatenate and re-project.

MultiHead(Q,K,V) = Concat(head₁,…,head₈) Wᴼ where headᵢ = Attention(Q Wᵢᴼ, K Wᵢᴷ, V Wᵢᵛ)

Why? A single head averages over all positions, losing the ability to attend to different things simultaneously. Multiple heads can each specialise. Click a head to see its likely role.

Click a head above to inspect it

Three uses of multi-head attention in the Transformer

Encoder self-attn

Each position attends to all positions in the previous encoder layer

Decoder masked self-attn

Each position attends only to earlier positions (future masked)

Cross-attention

Decoder queries attend over entire encoder output

3.5 Positional encoding

No recurrence, no convolution → the model is order-agnostic. Fix: add a positional encoding vector to each input embedding before the first layer.

PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Each position gets a unique d_model-length vector. The sinusoidal frequencies span 2π to 10000·2π. For any fixed offset k, PE_{pos+k} can be expressed as a linear function of PE_{pos} — so the model can learn to attend by relative position.

Learned vs sinusoidal

5 Training

Data

EN-DE: 4.5M sentence pairs, byte-pair encoding, 37K token vocab

EN-FR: 36M sentences, 32K word-piece vocab

Hardware

8× NVIDIA P100 GPUs, single machine

Base: 100K steps (~12h). Big: 300K steps (3.5 days)

Learning rate schedule (warmup)

lrate = d_model^(−0.5) · min(step^(−0.5), step · warmup_steps^(−1.5)) warmup_steps = 4000

Linear increase for the first 4000 steps, then decrease proportional to inverse square root of step number.

Regularisation

Residual dropout

Applied to sub-layer outputs and embedding+PE sums. P_drop = 0.1 (base), 0.3 (big EN-FR)

Label smoothing

ε_ls = 0.1. Hurts perplexity but improves BLEU and accuracy — the model learns to be more uncertain.

Optimizer: Adam with β₁=0.9, β₂=0.98, ε=10⁻⁹.

Model variations (ablations)

Heads (A)

Single head is 0.9 BLEU worse. Too many heads also hurts. 8 is the sweet spot.

Key size (B)

Reducing d_k hurts quality — compatibility is non-trivial.

Model size (C)

Bigger models are better. d_model=1024 gives +0.4 BLEU over base.

Dropout (D)

Very helpful for preventing overfitting. P_drop=0 drops 1.2 BLEU.

English constituency parsing

Trained a 4-layer Transformer on Penn Treebank WSJ (~40K sentences) with no task-specific tuning. Achieved 91.3 F1 WSJ-only; 92.7 F1 semi-supervised.

7 Conclusion

The Transformer is the first sequence transduction model based entirely on attention, replacing recurrent layers with multi-head self-attention.

What the authors planned next

— Apply Transformers to images, audio, video

— Investigate local / restricted attention for large inputs
